# Mòdul 2: Projects i Artifacts

## Què són els Projects?
Espais organitzats de treball on pots definir context persistent, coneixement, documents i instruccions específiques.

### Funcionalitats destacades
- Documents i instruccions permanents.
- Arts (artifacts): codi, dashboards, docs, gràfics, etc., que pots versionar i compartir.
- Project Knowledge Base: puja informació important i utilitza RAG per ampliar el context automàticament.

## Exemple pràctic: crear un Project
1. Nou Project → “Gestió Documental”
2. Instruccions:  
   – Sempre redacta en català formal.  
   – Revisa el document pujat i proposa títols i abstracts.

3. Puja un PDF i demana:  
   – “Resumeix en 5 punts i proposa millores estilístiques.”

## Tipus d’Artifacts
- **Codi execuble:** Python, JavaScript, HTML, etc.
- **Markdown/documentació:** Informes, manuals, cheatsheets.
- **SVG/gràfics:** Diagrames, visualitzacions, mapes.
- **Dashboards interactius** (React/HTML).

## Exercicis extra
1. Crea un Project per preparar una memòria TFG.
2. Genera i comparteix un Artifact de dashboard fictici amb 2 gràfics.
3. Prova modificar i versionar un Artifact: compara l’historial (remix).

## Q&A
- **Puc compartir un Artifact fora de Claude?**
  – Sí, pots publicar-lo i generar un enllaç exportable.
- **Què passa si pujo molta documentació?**
  – S'activa automàticament el mode RAG per gestionar context gran.
